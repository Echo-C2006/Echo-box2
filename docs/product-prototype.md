# 学科竞赛组队工具 —— 产品原型文档

> 本文档记录项目的产品定位、核心决策、页面流程与字段设计。  
> 最后更新：2026-05-20  
> 下次开发前请先阅读本文档恢复上下文。

---

## 一、产品定位

**一句话描述**：帮助在校学生找到学科竞赛队友，并提供组队后的轻量协作支持。

**目标用户**：在校学生（先本校试点，架构预留多校扩展能力）

**核心场景**：
1. 我想参加某竞赛，缺队友 → 去广场发帖或去人才库找人
2. 我很有能力，想被邀请 → 完善人才档案，等待被搜索
3. 队伍组好了 → 使用轻量队伍主页同步信息，重协作引导到外部工具（微信群、腾讯文档等）

---

## 二、关键决策（已确认）

| 维度 | 决策 | 备注 |
|------|------|------|
| **平台形态** | 响应式网页 | 手机 + 电脑兼顾 |
| **注册认证** | 开放注册 | 手机号/邮箱即可，门槛低 |
| **竞赛信息** | 内置竞赛库 | 发帖时必须从列表选择，保证信息规范 |
| **找人机制** | 广场 + 人才库 | 双向选择：既可刷招募帖，也可主动搜人 |
| **协作管理** | 轻量队伍主页 | 只做展示、公告、进度、外部链接，不做重协作 |
| **运营范围** | 本校试点 | 技术架构预留多校扩展能力 |
| **联系机制** | 站内申请/私信 | 保护隐私，流程规范 |
| **消息通知** | 浏览器 Web Push | 免费即时推送，需用户首次授权 |

## 四、核心页面详细定义

### 4.1 广场首页（默认首页）

**目的**：浏览招募帖，发现组队机会。

**布局**：
- 顶部：搜索栏 + 横向滚动的竞赛分类标签（全部 / 数学建模 / 挑战杯 / …）
- 次级：排序选项（最新发布 · 最热 · 即将截止）
- 主体：帖子卡片列表
- 空状态："暂无招募帖，来做第一个发帖的人吧"

**帖子卡片字段**：
- 竞赛名称（徽章样式，如蓝底白字"数学建模"）
- 帖子标题（一行截断，如"求两名队友，缺编程和论文"）
- 队伍状态标签：**招募中** / 已满
- 人员：已有 X 人 / 目标 Y 人
- 需求技能标签：Python · 数据分析 · 论文写作
- 发布时间（相对时间，如"3小时前"）
- 发帖人信息：昵称 + 年级专业
- 浏览量 / 申请数

**交互**：
- 点击卡片 → 帖子详情页
- 点击竞赛标签 → 筛选只显示该竞赛
- 下拉刷新 / 上拉加载更多

---

### 4.2 人才库页面

**目的**：按技能/意向主动找人。

**布局**：
- 顶部：搜索栏
- 筛选区（桌面端左侧常驻 / 移动端顶部"筛选"按钮弹出抽屉）：
  - 技能标签多选
  - 年级
  - 专业方向
  - 竞赛意向（来自竞赛库）
- 主体：人才卡片列表

**人才卡片字段**：
- 头像 + 昵称
- 年级 / 专业
- 技能标签（最多显示 3 个，超出+"n+"）
- 竞赛意向（"想找：数学建模、互联网+"）
- 个人简介（一行截断，如"熟用 Python，有两次美赛经验"）
- **邀请组队** 按钮

**交互**：
- 点击卡片 → 个人档案详情页
- 点击"邀请组队" → 弹出选择"从我的哪个队伍邀请" + 附言 → 发送站内申请
- 筛选条件实时生效

---

### 4.3 帖子详情页

**目的**：了解招募详情，申请加入。

**字段**：
- 竞赛名称（大标题旁徽章）
- 帖子标题
- 发帖人信息卡片（头像、昵称、年级专业、点击跳转其人才档案）
- 详细描述（正文，支持换行，MVP 先纯文本）
- 队伍现有成员列表（头像、昵称、角色：队长/队员、承担技能）
- 还缺：X 人，需求技能
- 队伍状态：**招募中** / 已满 / 已结束
- 帖子有效期（如"还有 5 天截止"）

**操作区**：
- **申请加入**（大按钮，已满或自己已申请则变灰/变状态）

**交互**：
- 点击"申请加入" → 弹出输入框：**申请理由**（50-200 字，如"我有 Python 经验，会爬虫和数据分析"）
- 提交 → 申请者看到状态变为"已申请，等待回复"
- 队长实时收到 **Web Push** + 站内消息
- 帖子满员后自动显示"已满"

---

### 4.4 个人档案页（人才详情）

**目的**：展示个人能力，被他人发现或查看队友背景。

**字段**：
- 头像、昵称
- 基本信息：年级、专业
- 技能标签（可多选 + 自定义输入）
- 竞赛经历（文本，如"2024美赛H奖"）
- 竞赛意向（来自竞赛库，标签展示）
- 可投入时间（如"每周 10 小时" / "暑假全职"）
- 个人简介
- **发私信** / **邀请组队** 按钮（自己看自己时变为"编辑资料"）

**交互**：
- 他人视角：可见所有信息，可操作私信/邀请
- 自己视角：可编辑所有字段
- 首次注册后弱引导："完善档案可提升被搜索到的概率"

---

### 4.5 发布招募帖流程

**入口**：顶部/底部 **+发帖** 按钮。

**步骤表单**：
1. **选择竞赛**（必填）→ 从内置竞赛库搜索/选择
2. **帖子标题**（必填，如"数学建模求队友，缺编程位"）
3. **详细描述**（选填，队伍现状、分工计划、对队友的期望）
4. **需求技能**（多选标签，如 Python、MATLAB、论文写作）
5. **人员设置**：已有几人 / 共需几人（自动计算"还缺几人"）
6. **有效期**：7 天 / 14 天 / 30 天（默认 14 天，到期自动下架）

**提交后**：
- 帖子进入广场列表
- 同时自动生成一个**初始队伍主页**（队长就是发帖人，队伍名默认=帖子标题）

---

### 4.6 站内申请与私信系统

#### A. 申请加入流程
- **申请者**：在帖子详情填写理由 → 提交 → 在个人中心"我的申请"查看状态（待回复 / 已通过 / 已拒绝）
- **队长**：收到 Web Push → 进入消息中心/帖子管理 → 查看申请人档案 → **同意 / 拒绝 / 先私信聊聊**
- **队长点击"同意"** → 申请者加入队伍，帖子"已有X人"自动 +1，满员则状态变"已满"

#### B. 私信聊天页
- **入口**：底部"消息"Tab
- **会话列表**：头像、昵称、最后一条消息预览、未读红点
- **聊天页**：
  - 上方显示对方档案入口
  - 消息气泡（纯文本，MVP 暂不支持图片/文件）
  - 快捷短语："你好，我对你的招募很感兴趣"、"请问还有名额吗？"
  - 输入框 + 发送

---

### 4.7 队伍主页（轻量协作中心）

**目的**：组队成功后的信息同步与轻量管理。

**入口**：帖子详情"进入队伍主页"、个人中心"我的队伍"。

**字段与模块**：
- **队伍信息区**：
  - 队伍名称（队长可编辑）
  - 参赛竞赛
  - 当前状态（招募中 / 备赛中 / 比赛中 / 已完赛）— 队长可更新
  - 比赛倒计时（如"距提交还剩 12 天"，手动设置或关联竞赛库）
- **成员列表**：
  - 头像、昵称、角色（队长/队员）、入队时承担的技能标签
  - 队长可移除成员、转让队长
  - 队员可"退出队伍"
- **队伍公告**（队长可编辑，纯文本，如"本周六下午3点图书馆讨论"）
- **外部协作区**（队长可添加/编辑）：
  - 微信群二维码图片
  - 腾讯文档 / Notion / GitHub 等链接
  - 提示："具体协作请跳转外部工具"
- **当前阶段/进度**（队长可更新，如"选题阶段 → 建模阶段 → 写作阶段"）

**权限表**：

| 操作 | 队长 | 队员 | 外人 |
|------|------|------|------|
| 看成员 | ✅ | ✅ | ✅ |
| 看外部链接 | ✅ | ✅ | ❌ |
| 编辑公告 | ✅ | ❌ | ❌ |
| 管理成员 | ✅ | ❌ | ❌ |
| 退出队伍 | — | ✅ | — |

---

### 4.8 注册 / 登录

**注册**：
- 邮箱 / 手机号
- 验证码（邮箱链接 / 短信，MVP 可先简化为邮箱+密码注册，后续加验证）
- 设置密码
- 昵称
- 注册后引导完善人才档案（可跳过）

**登录**：
- 账号（邮箱/手机号）+ 密码
- 记住我
- 找回密码

**首次登录后**：
- 浏览器弹出授权：**是否开启消息推送？**（说明：开启后才能收到组队申请提醒）

---

### 4.9 通知中心

**入口**：导航栏铃铛 / 消息 Tab 顶部分类切换。

**通知类型**：
- 📩 **申请类**："XX 申请加入你的 XX 队伍" / "你的申请已被通过"
- 💬 **私信类**："XX 发来新消息"
- 📢 **系统类**："你的招募帖将在 2 天后过期"、"你已被设为队长"

**交互**：点击通知直接跳转对应页面（帖子详情/聊天页/队伍主页）。

## 五、响应式关键差异

| 场景 | 桌面端 | 移动端 |
|------|--------|--------|
| 广场列表 | 2-3 列卡片网格，左侧可放筛选 | 单列卡片流，下拉刷新 |
| 人才库筛选 | 左侧边栏常驻 | 顶部按钮弹出底部抽屉 |
| 发帖表单 | 分栏布局，右侧实时预览 | 纵向单页滚动 |
| 队伍主页 | 多栏：左信息、中成员、右公告 | 纵向堆叠，Tab 切换模块 |
| 私信聊天 | 左右分栏：左会话列表、右聊天 | 全屏聊天页，返回会话列表 |

## 六、MVP 范围建议

### 第一阶段必须包含
- [ ] 注册登录（邮箱/手机 + 密码）
- [ ] 广场列表 + 发帖 + 帖子详情
- [ ] 人才库列表 + 个人档案创建/编辑/查看
- [ ] 站内申请加入队伍流程
- [ ] 私信聊天系统
- [ ] 队伍主页（信息展示 + 公告 + 成员管理）
- [ ] Web Push 通知接入
- [ ] 内置竞赛库（硬编码或管理员后台录入）

### 后续迭代再考虑
- [ ] 用户评价/信用体系
- [ ] 竞赛报名截止日期自动同步
- [ ] 高级筛选（按"每周可投入时间"筛选）
- [ ] 私信发送图片/文件
- [ ] 富文本编辑器（帖子、公告先纯文本）
- [ ] 多校切换与学校认证
- [ ] 帖子搜索（全文检索）

## 七、待决策事项（下次讨论）

1. **技术栈选型**：前后端分别用什么？数据库用 MySQL/PostgreSQL/MongoDB？
2. **部署方式**：服务器、域名、SSL？
3. **竞赛库维护**：初期硬编码一个常见竞赛列表？还是做简单的管理员后台录入？
4. **用户上传**：头像上传接哪个图床/云存储？微信群二维码图片怎么存？
5. **冷启动**：怎么让本校第一批学生知道这个网站？
